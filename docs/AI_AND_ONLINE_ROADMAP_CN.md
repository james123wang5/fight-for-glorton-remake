# 全角色/地图 AI 与公网对战路线

## 当前冻结基线

- 21 级：保留先前人工认可的 v5 基线，不用本轮较差候选覆盖。
- 22 级：冻结 2026-07-16 人工实战认可的第 1 轮候选。
- 当前训练域：Peach vs Peach、Mogadishu。跨角色或跨地图表现只能算泛化测试。
- 普通 `python play.py` 与训练环境隔离；新训练只能生成候选，不能自动替换正式模型。

## 全角色 AI 不应直接复制桃子模型

桃子的“追击、威胁判断、回场、边界意识、出招目的”可以共享，但枪、火箭、重量、
攻击帧、命中盒和移动能力属于角色适配层。推荐把策略拆成三层：

1. 通用目的层：追击、控场、近战、拦截、回场、躲避、拾取道具。
2. 角色动作适配层：把目的转换成该角色此刻合法的真实按键组合。
3. 地图导航层：平台连通图、跳跃落点、墙角脱困、死亡边界与移动机关预测。

这样可以复用桃子学到的战斗意识，同时避免让 Trash 或 Coffee 硬套桃子的火箭条件。

## 推荐训练顺序

### 阶段 A：补齐数据接口

- [x] 观察中加入角色类型、各动作可用性、剩余帧、地图编号和当前平台节点。
- [x] 每个角色建立动作能力表与专项测试：地面手刀、空中手刀、方向手刀、普通特殊、
  上特殊、回场、抓取和投掷。
- [x] 四张地图生成平台导航图；Mogadishu 的大地板会按实体建筑切开，移动平台按当前
  相位重新生成节点与边。
- [x] 为并发任务建立独立种子、模型目录、日志目录、场景清单和排他锁，避免多个训练
  进程互相覆盖。

阶段 A 已于 2026-07-16 完成，技术说明和检查命令见
[`ROSTER_AI_PHASE_A_CN.md`](ROSTER_AI_PHASE_A_CN.md)。墙角停滞、无意义跳跃和重复下落
属于奖励与评估规则，留在阶段 B 按角色接入，不能只凭平台图判定。

### 阶段 B：逐角色单图课程

仍固定 Mogadishu，每次只训练一个新角色：先对冻结 Peach 21/22，再做同角色镜像对战。
每个角色分别产出候选 21/22；通过专项考试和人工实战后才进入冠军池。

推荐顺序是 Default、Trash、Coffee、SBL、Aubergine。先从动作结构接近、回场容易验证的
角色开始，最后处理弹道或体型差异最大的角色。

### 阶段 C：逐地图课程

冻结角色冠军，依次加入 Rooftop、B52、Space。训练局按“单图专项 → 两图混合 → 四图混合”
推进，防止一次随机全部地图导致策略只学到保守的平均行为。

### 阶段 D：联赛与防遗忘

每轮训练 21 时冻结 22，再用新 21 训练 22；对手池同时保留最近冠军、历史快照、主动探针、
回场探针和真人录像策略。抽样建议以当前同角色对手为主，辅以跨角色与跨地图局。

正式评估矩阵至少覆盖：

```text
6个角色 × 4张地图 × 左右出生位 × 5类对手 × 多个固定种子
```

自动门槛包括胜率、决定局比例、远距离空转、墙角停滞、有效命中率、空中追击、回场成功率、
动作熵和左右出生位差异。最终仍保留人工验收，因为“很强但不像真人”不能只靠胜率发现。

## 真人对战继续学习

真人实战时不要直接在线更新正式 22 级。更安全的流程是：

1. 每局保存观察、合法动作掩码、真人输入、AI 输入、命中和最终结果。
2. 先用真人输入做行为克隆，得到独立的 `human_candidate`。
3. 把候选加入自对战对手池，用 PPO 微调，而不是逐帧照抄玩家。
4. 与冻结冠军做固定种子评估，再人工试玩；合格才晋升。

这既能学习真人连招和追击节奏，也不会因为一两局玩家故意乱按而破坏正式模型。

## 公网网页双人对战

当前网页/PWA 是静态客户端；把它放到 GitHub Pages 只能让朋友打开游戏，不能自动产生
远程房间。真正的远程对战还需要一个持续运行的权威服务器。

推荐最小版本：

1. 房间码：创建/加入，不做复杂账号。
2. 客户端只发送带序号和逻辑帧号的按键边沿。
3. 服务器以 25 ms / 40 Hz 运行 `BattleSimulation.step(inputs)`，负责随机种子、命中、
   掉命和胜负。
4. 服务器每 100–200 ms 发状态快照；客户端本地预测自己、插值显示远端。
5. 哈希不一致时平滑校正；短线保留房间并支持重连。
6. 基础版稳定后再加输入缓冲、延迟显示和回滚，不在第一版同时实现全部复杂功能。

部署结构建议保持前后端分离：静态 PWA 继续由 Pages/CDN 承载；Python `asyncio` WebSocket
服务器放在支持长连接的容器平台。这样朋友只需要打开网址和输入房间码，你的 Mac 可以
关机，训练环境也不会进入线上服务器。

上线前必须验证：相同种子/输入的跨机器摘要一致、房间互不串状态、恶意高频输入限流、
断线清理、版本不一致拒绝入房，以及至少 100–200 ms 人工延迟下仍可正常完成对局。
